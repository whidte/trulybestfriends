# Truly Best Friends Forever

> 让你的宠物永远陪伴在身边 —— 一个面向 Minecraft 1.21.1 (NeoForge) 的宠物管理模组。

驯服过的宠物会被自动追踪并持久化保存。即使宠物所在的区块未加载、宠物死亡、甚至存档切换维度，你都可以通过一个独立的宠物管理面板随时把它们召回身边、复活、或传送到它们的位置。

## 核心特性

- **自动追踪与持久化**：驯服任意可归属实体（狼、猫、马、鹦鹉、模组宠物等）后自动登记，NBT 数据按 `<存档>/trulybestfriends/<玩家UUID>/<宠物UUID>.nbt` 持久化，并维护一份 `pets_index.nbt` 索引。
- **宠物管理面板**：在物品栏中新增「宠物伙伴」标签页（基于 L2Tabs），也可通过自定义快捷键（默认未绑定，请在「选项 → 控制」里设置）打开。面板内可查看每只宠物的名称、种族、生命、所在维度与坐标。
- **实时数据同步**：服务器定期将选中宠物的最新状态（生命值、坐标、维度等）推送到客户端，GUI 内的血条与位置信息会自动刷新，无需重新打开面板。
- **收回 / 放出**：单击将宠物收回存储（带距离限制与冷却），再次单击放出。区块未加载时支持「召唤到玩家」机制，待加载完成后自动传送。
- **范围收回**：按住 **Shift** 单击收回按钮，可一次性收回范围内的多只宠物，**滚轮**可调节范围（1~16 格，默认 8）。
- **跨维度召唤**：支持从任意维度召回宠物到玩家身边，自动处理区块强制加载与实体去重，避免召唤出重复实体。
- **传送**：点击坐标可传送到宠物身边；也可将宠物传送到玩家身边。
- **发光定位**：给选中的宠物施加 5 秒发光效果，方便在复杂地形里找到它。
- **死亡复活**：被追踪的宠物死亡时掉落物会被清空（保留其 NBT），随后可在面板内用配置的复活物品（默认 1 个**不死图腾**）将其复活。复活时附带不死图腾的状态效果（生命恢复、伤害吸收、火焰抗性）。可在配置中将特定实体类型加入白名单，使其保留掉落物且不可复活（默认包含 `touhou_little_maid:maid`）。
- **两步删除确认**：为防止误操作，删除宠物数据需要两步确认 —— 先左键点击进入「待删除」状态，再 Shift+左键点击确认删除。切换选中实体时待删除状态会自动重置。
- **肩上宠物跟踪**：自动跟踪站在玩家肩上的鹦鹉等实体，下肩时自动重新关联 UUID，避免「肩上下来的宠物丢失追踪」。
- **优先级**：每只宠物可设置 1~6 级优先级，用于面板内排序与未来编队功能。
- **马匹背包备份**：对带容器的实体（如驮箱的马），独立备份其物品栏 NBT，兼容任意槽位数量的模组实体。
- **多语言维度名**：内置 20+ 常见维度（原版 3 维度 + 暮色森林、天境、彼岸、永昼/永曦之地、地下花园、蜜蜂领域、热带世界、盖亚维度、午夜维度、Ad Astra 全系列星球与轨道）的中英文显示名，并支持在配置中自定义。
- **NBT 离线编辑**：服务器关闭时可直接修改存档目录下的 `.nbt` 文件，下次启动时修改会被保留并应用到实体上（仅对已收回的实体推荐使用）。

## 安装

1. 安装 **Minecraft 1.21.1** + **NeoForge 21.1.228** 或以上版本。
2. 将以下模组 jar 放入 `.minecraft/mods/`：
   - Truly Best Friends Forever（本模组）
   - 可选：L2 Library 3.0.8+（提供物品栏标签页；未安装时使用快捷键）
   - 可选：Curios 9.5.1+（保存和恢复宠物饰品）
3. 启动游戏，进入存档即可，驯服一只宠物后即开始追踪。

> 服务端/客户端均需安装本模组及其依赖。本模组包含网络同步逻辑，纯客户端无法工作。

## 配置

配置文件位于 `config/trulybestfriends-common.toml`：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `syncIntervalTicks` | `103` | 全量扫描所有已加载归属实体并同步回磁盘的间隔（tick） |
| `localSyncIntervalTicks` | `5` | 扫描已完成「真正的好朋友」进度的玩家附近实体的间隔（tick） |
| `savePetDataCooldownTicks` | `100` | 将缓存中的宠物数据刷写到磁盘的间隔（tick）。玩家登出与服务器关闭总是立即刷写 |
| `recallRange` | `16.0` | 收回宠物的最大距离（格），`-1` 为无限 |
| `recallCooldownMs` | `3000` | 收回/召唤操作冷却（毫秒，最低 250） |
| `maxPets` | `64` | 单名玩家最多追踪的宠物数（1~128） |
| `areaRecallDefaultRange` | `8` | 范围收回的默认半径（1~16，游戏中可用滚轮调整） |
| `maxPendingSummons` | `6` | 区块未加载时每名玩家可排队的召唤数上限（+2 缓冲，未来也将作为编队模式的上限） |
| `reviveItem` | `minecraft:totem_of_undying` | 复活宠物所需物品 ID |
| `reviveItemCount` | `1` | 复活所需物品数量 |
| `reviveCooldownSeconds` | `120` | 复活冷却时间（秒，0~86400） |
| `noReviveWhitelist` | `["touhou_little_maid:maid"]` | 保留掉落物且不可复活的实体类型列表 |
| `autoRegisterBlacklist` | （见配置） | 不自动登记为宠物的实体类型列表，支持 `namespace:*` 通配符（默认屏蔽 Iron's Spells 的召唤物） |
| `enableLoginLoadDiagnostics` | `false` | 登录时校验宠物 NBT 文件并打印日志（调试用） |
| `clearOnDeathWhitelist` | （见配置） | 死亡时保留掉落物，并彻底清除追踪数据的实体类型列表 |

## 数据存储位置

所有数据保存在存档目录下，删除存档会一并删除宠物数据，删除本模组不会损坏存档：

```
<存档>/trulybestfriends/
├── pets_index.nbt              # 宠物索引（按实体类型分组）
└── <玩家UUID>/
    └── <宠物UUID>.nbt          # 每只宠物的完整 NBT
```

> 服务器关闭时可直接编辑 `<宠物UUID>.nbt` 文件来修改已收回宠物的属性（生命、名称、MaxHealth 等），下次启动时修改会通过 `entity.load(nbt)` 应用到召唤出的实体上。注意 `Pos` 与 `Dimension` 会在召唤时被召唤定位逻辑覆盖。

## 兼容性

- **理论兼容**任何遵循 `OwnableEntity`（可归属实体）接口的模组宠物。
- 使用完整实体 NBT 保存与恢复宠物；原版马匹容器使用公开 API，模组容器通过 NeoForge ItemHandler Capability 兜底，无需反射实体字段。
- 安装 Curios 9.5.1+ 后，宠物饰品会随召回和复活流程备份、恢复，并避免死亡掉落复制。
- 安装 L2 Library 3.0.8+ 后，宠物面板会注册到 L2Tabs 物品栏标签组。
- 已默认将东方女仆（`touhou_little_maid:maid`）加入不可复活白名单，因其死亡逻辑与本模组复活机制冲突。
- 已默认将 Iron's Spells 的召唤物（幽灵战马、召唤 vex / 僵尸 / 骷髅 / 北极熊 / 各类召唤武器等）加入自动登记黑名单，避免临时召唤物被误登记。
- 若某模组宠物的死亡掉落不应被清空，请将其实体 ID 加入 `noReviveWhitelist`。
- 若某模组的「可归属实体」不应被本模组追踪（如临时召唤物），请将其实体 ID 或 `命名空间:*` 加入 `autoRegisterBlacklist`。

## 已知限制

- 被追踪的宠物在死亡前若未被本模组同步保存（例如驯服后立刻死亡且未经过一个同步周期），可能无法复活。
- 范围收回不区分敌我，范围内的所有自身宠物都会被收回。
- 多人服务器上，跨维度的宠物召唤会受 `maxPendingSummons` 限制。
- 直接将 NBT 中的 `Health` 改为 `0` 后召唤，实体会进入「死亡但未收回」状态，召唤键将失效，需通过复活功能恢复。
